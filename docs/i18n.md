# Internationalization (i18n)

ExcaliDash's interface ships in **English (default)** and **Brazilian Portuguese
(pt-BR)**. Users switch language from a language button available on the login,
dashboard (sidebar), and settings screens. The choice is persisted to
`localStorage`.

This translates ExcaliDash's **own UI** (login, dashboard, settings, API
Keys/MCP, save status, etc.). It is separate from the Excalidraw **canvas**
language (the drawing toolbar), which is handled by `@excalidraw/excalidraw` and
its own selector. User-created content inside drawings is never translated.

## Stack

- [`i18next`](https://www.i18next.com/) + [`react-i18next`](https://react.i18next.com/)
- `i18next-browser-languagedetector` (order: `localStorage` → `navigator`)
- Default language: `en`. Supported: `en`, `pt-BR`.
- localStorage key: `excalidash-ui-lang` (distinct from the Excalidraw canvas key).

## Layout

```
frontend/src/i18n/index.ts            i18next init + config
frontend/src/i18n/locales/en.json     English catalog
frontend/src/i18n/locales/pt-BR.json  Brazilian Portuguese catalog
frontend/src/components/LanguageSwitcher.tsx   app-shell language button
```

## Using translations in components

```tsx
import { useTranslation } from 'react-i18next';

function Example() {
  const { t } = useTranslation();
  return <h1>{t('settings.title')}</h1>;          // "Settings" / "Configurações"
}
```

Interpolation uses i18next syntax:

```tsx
t('dashboard.empty.noSearchResults', { search });  // No results for "{{search}}"
```

## Adding or editing strings

1. Add the key to **both** `en.json` and `pt-BR.json` (keep the structures in sync).
2. Replace the literal in the component with `t('your.key')`.
3. Keep code, commands, and the literal API token placeholder
   `exd_REPLACE_WITH_YOUR_API_KEY` untranslated.

## Adding a language

1. Create `frontend/src/i18n/locales/<code>.json` mirroring `en.json`.
2. Register it in `frontend/src/i18n/index.ts` (`resources` + `supportedLngs`)
   and in `SUPPORTED_LANGUAGES`.

## Translated surfaces

Login / onboarding, dashboard (views, search, empty states, primary actions),
sidebar navigation, settings (incl. the Language card), API Keys / MCP headings,
the editor **save status** indicator, and common buttons/labels. The catalogs
also include keys for history, uploads, and errors that components can adopt
incrementally.
